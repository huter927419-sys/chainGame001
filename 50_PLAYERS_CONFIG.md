# 50玩家配置总结

## ✅ 已完成的配置

### 1. 玩家数量上限：50人

**优势：**
- ✅ 排名计算成本极低：~500 gas（vs 300玩家的3,000 gas）
- ✅ 获胜概率高：6%（vs 300玩家的1%）
- ✅ 游戏体验好：竞争适中，参与机会充足
- ✅ 适合初期运营：成本低，风险小

### 2. 可配置初始价格

**已实现功能：**
- ✅ `set_base_price`：设置基础价格（0.1-2 TON）
- ✅ `set_max_price`：设置价格上限（>=基础价格，<=10 TON）
- ✅ `getBasePrice()`：查询基础价格
- ✅ `getMaxPrice()`：查询价格上限

**默认值：**
- 基础价格：1 TON
- 价格上限：5 TON

### 3. 动态分配优化

**50玩家时的分配：**
- 奖池：75%（提高，弥补奖池规模小）
- 社区池：15%
- 预留池：10%

**理由：**
- 玩家少时，提高奖池比例可以吸引玩家
- 弥补奖池规模小的不足

## 💡 使用建议

### 初期运营
- 使用低门槛配置（0.5 TON基础价格）
- 吸引玩家参与
- 建立用户基础

### 稳定运营
- 使用标准配置（1 TON基础价格）
- 平衡成本和收益
- 保持玩家活跃度

### 大型活动
- 可以提高基础价格到1.5-2 TON
- 提高奖池吸引力
- 增加运营收益

## 🔧 配置方法

### 设置玩家上限
```tact
receive("set_max_players", 50)
```

### 设置基础价格
```tact
// 设置为0.5 TON
receive("set_base_price", 500000000.toCoins())

// 设置为1 TON（默认）
receive("set_base_price", 1000000000.toCoins())
```

### 设置价格上限
```tact
// 设置为3 TON
receive("set_max_price", 3000000000.toCoins())

// 设置为5 TON（默认）
receive("set_max_price", 5000000000.toCoins())
```
